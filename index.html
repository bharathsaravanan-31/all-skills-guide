Support & Investigation
3 skills
Investigate Support Cases
/investigate
End-to-end MyGeotab support case investigation. Routes automatically to the right branch — odometer, OEM, vehicle data, or HOS — based on the case details.
When to useOdometer issues, IFTA mileage discrepancies, odometer drops, negative odometer, HOS violations, HOS ruleset questions, engine data issues, or protocol problems. Accepts SF case number, serial number, database URL, customer name, or vehicle make.
Example
/investigate 01721196
/investigate G9RTRMN6VSMZ my.geotab.com/acmecorp
/investigate Kenworth odometer drop
Investigate OEM Devices
/investigate-oem
Deep-dive 9-step OEM device troubleshooting — enrollment errors, data not flowing, activation failures, and data discrepancies for embedded OEM devices.
When to useOEM device serial (prefix: DW, EB, E5, CO, FT, D8, DY, E1, JJ…) or a 17-char VIN. Use when general /investigate isn't enough for OEM-specific issues.
Example
/investigate-oem DW1234567890
/investigate-oem 1HGBH41JXMN109186
/investigate-oem 01721196
Geotab Resource Fetcher
/util-geotab-onesource
Fetches and searches Geotab resources asynchronously across 5 sources simultaneously: community forum, SDK API docs, Video Hub, MyGeotab API, and internal docs.
When to useLooking up SDK methods, community forum answers, training videos, or internal documentation. Faster than searching each source manually.
Example
"Search Geotab resources for GO9 firmware update"
"Find SDK docs for GetFeed method"
"Look up community posts about odometer calibration"

Auth & Utilities
2 skills
Unified Auth Manager
/unified-auth-manager
Single CLI to manage encrypted credentials and live sessions for JIRA, MyGeotab, Google, and Salesforce APIs. Handles token refresh, session validation, and secure storage.
When to useSetting up API access, checking auth status across tools, or managing tokens before running any multi-platform investigation. Say "uam", "unified auth", or "login to [platform]".
Example
"Check my auth status"
"Login to Jira and MyGeotab"
"Manage my API tokens"
/unified-auth-manager
Support Workflow Agent
/workflow-support-agent
Multi-agent support query system that routes questions across Salesforce cases, Jira ESUPSD tickets, Google Drive KB, and Google Chat discussions — returning synthesized answers from all sources.
When to useComplex support questions that need cross-platform research. Automatically searches the right sources and synthesizes findings instead of checking each tool manually.
Example
"Find everything about this customer's issue across all our tools"
"Search KB, Jira, and Salesforce for [topic]"
/workflow-support-agent

Geotab Vitality — Customer Success
11 skills
Log Activity to CRM
/vitality-log-crm
Converts rough post-call notes into a structured Salesforce-ready activity log. Handles batch mode for multiple calls at once.
When to useAfter any customer call, onboarding session, check-in, or enrollment follow-up.
Example
"Log this call — spoke with Jane at Acme, device arrived,
privacy concern raised, next: send FAQ by Friday"
Adapt Content Across Platforms
/vitality-adapt
Takes one Vitality message and outputs Slack post + manager email + intranet article + slide talking points simultaneously — no rewriting.
When to useWhen the same update needs to go to multiple channels at once.
Example
"We hit 42% enrollment, up from 30%. Target 60% by June.
Adapt into Slack post, manager email, 3 slide talking points."
Analyze Patterns in Feedback
/vitality-analyze-feedback
Extracts themes, sentiment, frequency counts, and quick-win recommendations from surveys, support emails, and Slack feedback — all sources at once.
When to useAfter survey results land or when you have a batch of user feedback to process.
Example
"Here are 3 feedback sources from the past 60 days [paste].
Find top themes, sentiment breakdown, and 3 quick wins."
Generate Project Status Reports
/vitality-status-report
Consolidates Slack, email, and tracker updates into a structured weekly status report — enrollment progress, cohort health, blockers flagged, executive summary.
When to useWeekly or monthly reporting to leadership. Say "generate status report" or "weekly Vitality update".
Example
"Here are updates from this week [paste Slack + email + tracker].
Generate status report with enrollment vs target and blockers."
Surface Themes from All Channels
/vitality-surface-themes
Cross-references ALL feedback sources and surfaces only validated themes — those appearing in 2+ channels. Separates real signal from one-off noise.
When to useQuarterly synthesis to confirm what's genuinely widespread across the program.
Example
"Synthesize Q1 feedback from surveys, Slack, and CS notes [paste].
Show only themes in 2+ sources, ranked by priority."
Source Insights to Build a Deck
/vitality-build-deck
Synthesizes enrollment data, wins, feedback themes, and blockers into a slide-by-slide outline with talking points and a 2-paragraph executive narrative.
When to useQBRs, stakeholder presentations, enrollment milestone decks.

7 slides. Audience: leadership team."
Build a Daily Briefing
/vitality-daily-brief
Builds a structured morning briefing from pasted Slack, email, and tracker updates — urgent items, at-risk customers, overdue follow-ups, and one strategic focus.
When to useStart of every workday. Say "daily brief" or "what do I focus on today".
Example
